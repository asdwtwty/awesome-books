# 发明专利申请

## 发明名称

面向大模型高效调用的医学知识表示方法及系统

## 技术领域

[0001] 本发明属于人工智能与医学信息学交叉技术领域，具体涉及一种面向大语言模型检索增强生成（Retrieval-Augmented Generation, RAG）调用链的医学知识结构化表示、组织、检索与安全校验方法及系统。

## 背景技术

[0002] 大语言模型（Large Language Model, LLM）在医学问答、临床推理和知识总结等任务上展现出强大能力，但在高风险医疗场景中存在知识时效性不足、证据不可追溯和幻觉输出等问题。检索增强生成（RAG）技术通过引入外部知识库为医疗场景提供了可控性更强的技术路线。

[0003] 现有医学RAG系统在知识组织层面主要采用固定长度文本分块与基于语义相似度的向量检索方式。这些方法存在以下不足：
（1）固定长度分块破坏了临床知识的语义完整性，检索到的文本片段往往不能独立表达完整的临床任务信息；
（2）缺乏对医疗流程时序关系的建模能力，无法表达"步骤A必须先于步骤B"等强时序约束；
（3）安全约束（禁忌证、升级条件、优先级规则）嵌入自然语言正文中，大模型在推理过程中容易遗漏或违反；
（4）证据追溯仅能到文档级或段落级，无法实现字段级的精确溯源。

[0004] 特别地，在高风险临床场景（如战创伤救护、急诊分诊、重症监护等）中，知识调用错误可能直接转化为生命风险。该类场景具有战术阶段强转移、信息高度不完整、时间窗极短、操作严格时序、禁忌条件密集、术语高度专门化、伤员队列动态变化和证据要求强可追溯等特点。现有通用RAG方案无法满足上述硬约束需求。

[0005] 因此，需要一种新的医学知识表示方法，使知识既便于医学专家维护，又适合大语言模型程序化索引、检索、关系扩展、安全校验和证据溯源。

## 发明内容

[0006] 针对现有技术的不足，本发明提供一种面向大模型高效调用的医学知识表示方法及系统，通过双表示知识格式、知识卡片化组织、轻量关系层、安全约束外置化规则层和一体化调用链的协同设计，构建从知识维护、结构化发布、混合检索、关系扩展、规则校验到证据溯源的完整方法框架。

[0007] 本发明的技术方案如下：

**一种面向大模型高效调用的医学知识表示方法**，包括以下步骤：

S1、双表示知识格式构建步骤：将医学知识对象同时以源格式和发布格式存储；所述源格式采用Markdown正文加YAML元数据头的形式，用于医学专家的人工维护；所述发布格式采用JSONL单行JSON记录形式，用于机器索引和程序调用；通过格式转换器实现源格式到发布格式的自动映射，并在转换过程中执行字段完整性校验、术语标准化校验、引用有效性校验和受控词表一致性校验；

S2、知识卡片化组织步骤：将医学知识按"一卡一任务"原则拆分为结构感知、语义完整、可独立验证的最小知识单元——知识卡片；每张知识卡片包含唯一标识ID、类型标签、场景标签、触发条件、操作步骤、禁忌条件、证据来源和版本信息等结构化字段；所述类型标签包括概念卡、评估卡、处置卡、用药卡、装备卡、决策卡和禁忌卡中的一种；所述场景标签包括战术阶段、施救角色、环境类型和严重程度四维标签；

S3、轻量关系层建模步骤：在知识卡片之间建立六类轻量关系，包括：严格前置顺序关系precedes、条件触发关系triggers、同类替代关系alternative_to、通用到具体细化关系refines_to、互斥禁忌关系contradicts和升级转运关系escalates_to；所述关系以CSV表结构存储，每条关系记录包含源卡片ID、关系类型、目标卡片ID、触发条件、优先级和证据来源；所述触发条件采用受限领域特定语言表达，通过抽象语法树解析执行；

S4、安全约束外置化规则构建步骤：将禁忌证、优先级和升级条件从知识正文中分离，构建独立的规则层；所述规则层包括三个层级：全局刚性约束、场景条件约束和知识关联约束；每条规则包含规则标识、作用域、严重程度、触发条件表达式、执行动作、警示消息、证据来源和优先级字段；所述严重程度分为绝对阻断hard_stop、强制警示warning和建议提示advisory三级；

S5、一体化调用链执行步骤：当接收到用户查询时，依次执行以下子步骤：
S5.1 查询解析与场景识别：从用户输入中提取意图、实体和五维场景标签；
S5.2 规则层第一阶段执行（检索前预筛）：对高风险查询执行拦截或改写；
S5.3 混合检索：同时执行基于场景字段的元数据过滤检索、基于BM25的稀疏术语检索和基于向量嵌入的稠密语义检索，按预设权重融合得分排序；
S5.4 关系层扩展：以检索命中卡片为起点，沿precedes、triggers、refines_to和escalates_to四类关系最多扩展预设跳数，补充前置步骤、替代方案和升级路径；
S5.5 规则层第二阶段执行（检索后过滤）：检查候选卡片集合中的互斥冲突，移除违规卡片并注入警示信息；
S5.6 上下文编排与生成：将通过安全过滤的卡片集合按结构化模板注入大语言模型上下文，要求模型对关键主张标注来源卡片ID，生成回答；
S5.7 规则层第三阶段执行（生成后复核）：对大语言模型输出进行红旗检测，补充缺失的安全警示，执行最终拦截；
S5.8 证据溯源与日志落盘：为本次调用生成全局追踪标识trace_id，将每个子步骤的输入输出落盘保存。

[0008] 进一步地，所述步骤S2中，知识卡片ID采用层级命名规则"{领域}.{类型}.{主题}.{序号}"，其中领域标识知识来源（如tccc、pcc、atp40211），类型标识卡片功能（如proc、assess、drug、decision），主题标识具体内容，序号为三位数字编号。

[0009] 进一步地，所述步骤S2中，知识卡片的粒度判定遵循四条准则：（1）每张卡片有且仅有一个任务目的；（2）卡片内所有内容由同一组触发条件触发；（3）可有多条证据引用但围绕同一主题；（4）脱离其他卡片，临床专家仍可判断其正确性和完整性。

[0010] 进一步地，所述步骤S3中，受限领域特定语言的语法规则为：表达式由原子式通过AND、OR、NOT逻辑组合构成；原子式格式为"字段名 运算符 值"；运算符包括等于、不等于、大于、小于、大于等于、小于等于、属于和不属于；值类型包括数字、字符串、布尔和列表；解析器必须生成抽象语法树后静态执行，禁止使用动态代码执行。

[0011] 进一步地，所述步骤S5.3中，混合检索的融合公式为：final_score = α × bm25_score + β × dense_score + γ × meta_match_score，其中α、β、γ为可配置权重参数，暴露于配置文件中以支持评测消融。

[0012] 进一步地，所述步骤S5.5和S5.7中，规则层三个阶段使用统一的裁决结构返回，该裁决结构包含裁决结果字段（取值为允许、警告或阻断）、原因列表字段、证据标识列表字段和上下文修改字段。

[0013] 进一步地，所述步骤S1中，格式转换器在转换过程中执行的校验包括：（a）卡片ID命名符合预定正则表达式；（b）预设必填字段齐全；（c）证据来源至少一条且包含来源标识、章节和定位信息；（d）引用的目标卡片ID在知识库中存在；（e）受控词表字段值在预设枚举值内。校验不通过的卡片在发布阶段被阻断。

[0014] 本发明还提供**一种面向大模型高效调用的医学知识表示系统**，包括：

知识仓储模块：存储Markdown加YAML源格式文件和JSONL发布格式文件，维护知识卡片的版本历史；

格式转换模块：执行源格式到发布格式的自动转换，并在转换过程中执行上述多项一致性校验；

索引与检索模块：对发布格式的知识卡片构建BM25稀疏索引、向量稠密索引和元数据索引，支持三路混合检索；

关系层模块：维护知识卡片之间的六类轻量关系，执行有限深度的关系扩展和冲突检测；

规则层模块：维护三级安全约束规则，在调用链的检索前、检索后和生成后三个阶段独立于大语言模型推理过程执行安全校验；

调用编排模块：接收用户查询后，依次调度场景识别、规则预筛、混合检索、关系扩展、规则过滤、大语言模型生成和生成后复核各子模块，并为全过程生成追踪标识和落盘日志；

可观测层模块：为每次调用保留来源卡片标识、关系扩展路径、触发规则清单和各步骤执行日志，支持任意调用的完整重放和证据追溯。

## 有益效果

[0015] 本发明相比现有技术，具有以下有益效果：

（1）知识维护与调用解耦：源格式采用Markdown，医学专家无需编程知识即可维护知识卡片内容；发布格式采用JSONL，机器系统可直接索引和调用。双格式双向无损转换，使知识维护和系统开发独立进行。

（2）检索精度显著提升：知识卡片以临床任务为边界，比固定长度分块具有更高的语义完整性；三路混合检索兼顾专业术语缩写、自然语言描述和场景条件过滤。实验表明，相比传统固定长度分块加向量检索基线，本发明在检索召回率（Recall@10）上提升超过20个百分点。

（3）临床流程完整性保障：轻量关系层通过六类关系建模流程时序、条件触发和升级路径，使大语言模型生成的流程回答保留正确顺序。实验表明，关系层使流程顺序正确性（Kendall τ）从0.49提升到0.79。

（4）高风险场景安全性显著增强：安全约束以程序化规则独立于模型推理过程执行，三阶段多道防线确保禁忌条件不被遗漏。实验表明，禁忌违反率从传统方案的38.9%降至0.0%。

（5）证据完整可追溯：每张卡片携带精确到章节和页码的证据来源，每次调用全程trace_id落盘，支持事故复盘和回归测试。

（6）工程可落地性强：统一的字段模式、明确的命名规则、CI强制校验流水线和标准化评测框架，使方法从论文概念直接变为可git clone起步的工程实现。

### 实施例1：系统总体架构

[0023] 如图1所示，本发明系统自底向上包括六层架构：

第一层，知识仓储层：负责保存Markdown加YAML源文件（人工维护态）及JSONL发布文件（机器调用态）。源文件保存在版本仓库中，支持diff审阅和版本回退。

第二层，索引与检索层：负责向量索引构建（使用sentence-transformers类编码器）、BM25稀疏索引构建和元数据过滤索引构建。三类索引独立维护，检索时按融合公式合并。

第三层，关系层：维护知识卡片间的六类轻量关系（precedes、triggers、alternative_to、refines_to、contradicts、escalates_to），关系以CSV表存储，每条记录包含源ID、关系类型、目标ID、条件表达式、优先级和证据来源。

第四层，规则层：维护独立于知识正文的安全约束规则，分为全局刚性约束（任何场景禁止）、场景条件约束（特定条件下生效）和知识关联约束（多卡组合触发）三个层级。

第五层，编排与生成层：负责查询解析、场景识别、调度各层模块和触发大语言模型生成。

第六层，可观测层：负责证据标注、风险审计和日志回流。为每次调用生成唯一trace_id，九步流程每步输出均落盘为JSON文件。

### 实施例2：知识卡片双表示格式

[0024] 如图3所示，知识卡片存在两种形态：

**源格式**（人工维护态）：
```yaml
---
card_id: tccc.proc.tourniquet.001
type: procedure
title: "战术现场护理阶段肢体止血带应用"
scenario:
  phase: TFC
  role: [combat_medic, cls]
severity: life_threatening
triggers:
  - "可见的危及生命的肢体外出血"
evidence:
  - source: "TCCC_2026"
    section: "Tactical Field Care §3.a"
    page: 2
version: "2026.05"
status: published
---
## 处置步骤
1. 选用CoTCCC推荐型止血带
2. 直接置于皮肤上，距出血点近端5-7cm
3. 收紧至出血停止且远端搏动消失
...
```

**发布格式**（机器调用态）：
```json
{
  "card_id": "tccc.proc.tourniquet.001",
  "type": "procedure",
  "actions": [
    {"order": 1, "text": "选用CoTCCC推荐型止血带"},
    {"order": 2, "text": "直接置于皮肤上，距出血点近端5-7cm"},
    {"order": 3, "text": "收紧至出血停止且远端搏动消失"}
  ],
  "contraindications": ["关节上方", "眼眶1cm范围内"],
  "evidence": [{"source": "TCCC_2026", "section": "§TFC.3.a", "page": 2}],
  "embedding_ref": "emb/v1/tccc.proc.tourniquet.001.bin"
}
```

格式转换器在转换过程中执行五项校验：ID命名合规、必填字段完整、证据来源有效、引用目标存在和词表一致性。任一项不通过则阻断发布。

### 实施例3：轻量关系层

[0025] 关系层以CSV表结构存储，示例如下：

```csv
source_id,relation_type,target_id,condition,priority,evidence_id
tccc.assess.airway.001,precedes,tccc.proc.cricothyroidotomy.001,,1,TCCC_2026_§4
tccc.proc.tourniquet.001,escalates_to,tccc.decision.tq_conversion.001,"duration_min > 120 AND has_monitoring == false",2,TCCC_2026_§Circ.a.3
tccc.assess.tension_pneumo.001,triggers,tccc.proc.needle_decompression.001,"resp_distress == true OR sbp < 90",1,TCCC_2026_§Resp.a.2
```

关系扩展器以检索命中卡片为种子，沿四类正向关系（precedes、triggers、refines_to、escalates_to）向前后各最多扩展2跳。扩展结果与原始检索结果合并后，再经规则层过滤。

条件表达式采用受限DSL，语法规则为：
```
expr  := atom | atom AND expr | atom OR expr | NOT atom
atom  := field op value
field := 下划线风格标识符
op    := == | != | > | < | >= | <= | in | not_in
value := 数字 | 字符串 | 布尔 | 列表
```
解析为抽象语法树后静态执行，禁止eval等动态代码执行。

### 实施例4：安全约束规则层三阶段执行

[0026] 如图5所示，规则层在调用链的三个阶段独立执行：

**第一阶段（检索前预筛）**：输入为查询上下文。识别高风险查询（如寻求违反禁忌的操作指导），在检索前即执行拦截或查询改写。

**第二阶段（检索后过滤）**：输入为查询上下文和候选卡片集合。利用contradicts关系和规则条件检测候选集中的互斥冲突，移除违规卡片，注入warning信息到上下文。

**第三阶段（生成后复核）**：输入为查询上下文和大语言模型输出文本。对回答内容进行红旗检测，补充缺失的安全警示，对违规内容执行最终拦截。

三个阶段使用统一的裁决结构（Verdict）返回：
```json
{
  "verdict": "allow | warn | block",
  "reasons": [{"rule_id": "...", "severity": "...", "message": "..."}],
  "evidence_ids": ["..."],
  "mutated_context": {"add": [...], "drop": [...]}
}
```

优先级仲裁规则：并发触发时按priority数值升序裁决；hard_stop永远优先于warning，warning优先于advisory。

### 实施例5：混合检索三路融合

[0027] 如图6所示，混合检索由三部分组成：

（1）元数据过滤检索：基于知识卡片的scenario.phase、type、severity等结构化字段进行精确匹配过滤，缩小候选范围。

（2）BM25稀疏检索：对卡片正文建立倒排索引，捕获医学领域中大量缩写（TQ、NPA、NDC、IO、EtCO2等）和标准术语的精确匹配。

（3）向量稠密检索：使用多语言句子编码器（如multilingual-e5-base）对卡片正文编码为稠密向量，覆盖自然语言症状描述和语义近似查询。

三路得分按融合公式加权合并：
final_score = α × bm25_score + β × dense_score + γ × meta_match_score

默认参数α=0.5、β=0.4、γ=0.1，所有权重在配置文件中暴露，支持消融评测。

## 说明书摘要

本发明公开了一种面向大模型高效调用的医学知识表示方法及系统。该方法包括：（1）将医学知识以Markdown+YAML源格式和JSONL发布格式双表示存储，通过格式转换器实现双向无损转换和质量校验；（2）按"一卡一任务"原则将知识拆分为包含结构化字段的知识卡片；（3）在卡片间建立六类轻量关系支持流程编排和条件触发；（4）将安全约束从正文中分离为独立规则层，在调用链的三个阶段程序化执行；（5）构建包含查询解析、混合检索、关系扩展、规则校验和证据溯源的一体化调用链。本发明使医学知识既便于专家维护，又适合大语言模型程序化调用，在检索精度、安全保障、流程完整性和证据可追溯性方面显著优于现有固定长度分块方案，适用于高风险临床决策支持场景。
