# 创伤失血性休克 (T/HS) 数字孪生实施路线

## ——面向**临床决策支持** (Clinical Decision Support, CDS) 的工程化方案

> **应用目标**：临床/院前/战伤救治现场的实时决策支持，**不**用于机制研究。
> **架构**：`(1D 脉搏波 + 3D 局部出血损伤) → 0D 全身闭环 → 决策层`
> **核心追求**：实时性、可解释性、临床可验证、与监护设备无缝集成、明确的可操作输出。

---

## 1. 总体架构

```
┌────────────────────────────────────────────────────────────────────┐
│                                                                    │
│   ┌──────────────────────┐         ┌──────────────────────────┐    │
│   │  L1  1D 脉搏波        │         │  L0  3D 局部出血损伤       │    │
│   │  (ADAN / openBF)     │         │  (CFD/FSI 离线降阶)        │    │
│   │                      │         │                          │    │
│   │  输入:  心室射血、顺应性 │         │  输入: P_local(t)、损伤几何 │    │
│   │        (来自 L2)      │         │        干预参数(来自决策)   │    │
│   │  输出:  ABP/PPG 波形   │         │  输出: Q_bleed(t)         │    │
│   └──────────┬───────────┘         └────────────┬─────────────┘    │
│              │                                  │                  │
│              │ 监护波形                          │ 出血流率           │
│              │ (用于参数辨识/                    │ (作为失血输入)     │
│              │  数据同化)                        │                  │
│              ▼                                  ▼                  │
│   ┌────────────────────────────────────────────────────────────┐   │
│   │            L2  0D 全身闭环 (核心引擎，实时)                   │   │
│   │   状态:  V_blood, V_int, MAP, CO, HR, SVR, SpO₂, Hct …      │   │
│   │   控制:  压力反射 + RAAS/ADH + 呼吸                          │   │
│   │   输入:  Q_bleed, Q_resus(晶/胶/血制品), Q_drug(血管活性药)    │   │
│   └────────────────────────────────────┬───────────────────────┘   │
│                                        │                           │
│                                        │ 状态向量 + 预测轨迹        │
│                                        ▼                           │
│   ┌────────────────────────────────────────────────────────────┐   │
│   │            决策层 (Clinical Decision Support)                │   │
│   │  • 失代偿风险评分 / 心血管崩溃倒计时                           │   │
│   │  • 推荐复苏方案 (晶/胶/血制品流速、目标 MAP)                    │   │
│   │  • 允许性低血压安全窗口                                        │   │
│   │  • 止血/手术优先级建议                                         │   │
│   │  • 后送/转运时间窗预测                                         │   │
│   │  • 早期预警告警 (基于状态估计)                                  │   │
│   └────────────────────────────────────────────────────────────┘   │
│                                                                    │
└────────────────────────────────────────────────────────────────────┘
              ▲                                       │
              │ 实测信号 (ECG/ABP/PPG/SpO₂/EtCO₂)      │ 可执行医嘱/告警
              │ 干预记录 (输液速率、止血带等)             │
              │                                       ▼
        ┌──────────────────┐                ┌──────────────────┐
        │ 床旁监护 / 院前监护 │                │  临床医生 / 卫生员  │
        └──────────────────┘                └──────────────────┘
```

**关键设计原则**：
- **L2 是实时心脏**：所有决策基于 L2 的实时状态估计，必须能在 ≤1 s 内完成 1 步仿真。
- **L0、L1 是 L2 的"输入端工厂"**：L0 给出真实驱动的 `Q_bleed(t)`，L1 给出与监护信号匹配的波形特征；它们通过**离线降阶 + 在线查表/代理模型**保证实时性。
- **决策层只读取 L2 状态**：保持决策算法与建模解耦，便于独立验证、独立迭代。
- **数据同化双向闭环**：监护数据→L2 参数辨识→决策；干预记录→L0/L2→重新预测。

## 2. 各层职责（CDS 视角）

### 2.1 L0 — 3D 局部出血损伤模块

| 维度 | 内容 |
|---|---|
| **CDS 角色** | 把"**经验估计的出血率**"升级为"**由真实损伤驱动的出血率**"——这是 CDS 区别于普通监护的核心价值 |
| **不做** | 不在床旁实时跑 3D CFD（计算太贵，临床不可接受） |
| **做** | **离线**为典型损伤场景跑 3D-CFD/FSI，得到 `Q_bleed = f(P_local, 几何, 干预)` 的**降阶代理模型 (ROM)**；**在线**只做查表/神经网络推理 |
| **典型损伤场景库**（建议覆盖）| ① 四肢动脉损伤 (穿透/截肢) ② 躯干钝挫伤实质脏器 (肝/脾) ③ 骨盆骨折后腹膜出血 ④ 主动脉破裂/夹层 ⑤ 颈部血管损伤 ⑥ 腹腔内大血管损伤 |
| **干预参数化**（关键）| 止血带压力、止血敷料覆盖率、伤口填塞深度、压迫位置——必须把可干预变量显式建模 |
| **输出接口** | `Q_bleed(t) = ROM(P_local(t), injury_id, intervention_params)` |
| **降阶方法学**| 离线 CFD 网格化扫描 → POD/Krylov 降阶 或 神经网络 (类似 Rubio & Marsden 2024 在分叉处的做法) |
| **临床输入** | 现场快速评估：损伤分类（下拉）、估计破口大小（影像或目视）、是否已干预 |
| **不确定性处理** | 输出 `Q_bleed` 的均值 + 置信区间，用 Monte Carlo 或 ensemble 传到 L2 |

### 2.2 L1 — 1D 脉搏波模块

| 维度 | 内容 |
|---|---|
| **CDS 角色** | **数据同化的桥梁**——把床旁监护的真实波形 (ABP、PPG、SpO₂) 与模型预测对齐，反推 L2 的隐藏状态/参数 |
| **不做** | 不输出"诊断"——它的产出仅供数据同化用 |
| **做** | 接 L2 的心室射血波 + 动脉顺应性，沿动脉树传播到监护探头位置（手指 PPG、桡动脉 A-line），生成**与设备同位点的合成波形** |
| **关键应用** | (1) **早期失血检测**：脉搏波形特征（脉压、上升斜率、PWV）变化早于 MAP 变化 (2) **个体参数辨识**：同一患者 0D 模型有十几个未知参数（心室收缩性、外周阻力、顺应性…），L1 提供波形特征作为额外约束 (3) **传感器→模型的反向同化**：把可穿戴/无创设备的波形信号映射到 L2 状态 |
| **简化策略** | 不需要全 ADAN 2000 段——选**与监护点相关的子树**（30-100 段足够）以保证实时 |
| **求解器** | openBF (1D FV, 开源, 已验证) 或 first_blood (1D + 虚拟队列工具链) |
| **输出接口** | `(P_radial(t), P_finger(t), PPG(t)) = 1D_solve(P_aorta(t), C_arterial)` |

### 2.3 L2 — 0D 全身闭环（核心引擎）

| 维度 | 内容 |
|---|---|
| **CDS 角色** | **实时状态估计器 + 短期预测器**。所有决策都基于它的状态向量。 |
| **建模基础** | 推荐采用 **Jin & Reifman 2023 (74 ODE 战创伤 CR 模型)** 或 **Chalumuri & Bighamian 2024 (FDA OSEL 出血/复苏模型)** 作为内核——这两者已用动物-人多数据集验证，且包含组织间隙腔室（处理 transcapillary refill 必备） |
| **必须包含的子模块** | ① 心室 ESPVR/EDPVR (Frank-Starling) ② 系统 + 肺循环 ③ **组织间隙腔室** (Starling) ④ 压力反射 (HR、SVR、Venous tone) ⑤ RAAS/ADH 长时调控 ⑥ 简化呼吸/气体交换 (SpO₂、EtCO₂) ⑦ 血液稀释/Hct 动力学 |
| **输入接口** | `Q_bleed(t)` ← L0；`P_local(t)` → L0；`P_aorta(t)` → L1；`Q_resus(t), Q_drug(t)` ← 决策层/医嘱；监护信号 → 数据同化 |
| **输出接口** | 完整状态向量 → 决策层 |
| **实时性要求** | 单次正向仿真：< 100 ms / 仿真分钟（即仿真 30 min 的预测在 3 s 内完成）—— 集成步长 1–10 ms，硬件普通 CPU |
| **数据同化方法**| Unscented Kalman Filter (UKF) 或 Ensemble Kalman Filter (EnKF)，每 1–5 s 用监护数据更新一次状态 |
| **个体化** | 入院 30 s 内用 HR、MAP、波形特征做参数初始化；之后随监护数据持续在线辨识 |

### 2.4 决策层 — 临床决策支持

| 维度 | 内容 |
|---|---|
| **CDS 角色** | **唯一对接临床医生**的部分。把 L2 的数学状态翻译成**可操作建议** + **可解释证据** |
| **必须遵循的设计原则** | (1) 推荐 ≠ 强制：所有输出仅作建议，最终决策权属于医生 (2) 全过程留痕 (3) 给出置信度 + 关键证据变量 (4) 允许医生覆盖并记录原因 |
| **决策原语 1：状态评估** | 当前血容量丢失 % (估计) / 休克分级 (Class I-IV) / 失代偿风险评分 |
| **决策原语 2：短期预测** | "若不干预，预计 X 分钟内 MAP < 65 / 心血管崩溃" |
| **决策原语 3：复苏方案推荐** | 模型预测控制 (MPC)：在约束下 (止血未完成 → 允许性低血压；已止血 → 正常 MAP) 优化 `Q_resus` 时序，使预测轨迹达成目标 MAP |
| **决策原语 4：允许性低血压窗口** | 基于 L0 的 `Q_bleed = f(P_local)` 曲线，给出"MAP 维持在 Y mmHg 时，到达手术止血前可耐受的最长时间" |
| **决策原语 5：转运/后送决策** | 综合短期预测 + 可用资源 → 评估"转运 X 分钟到上级救治机构" 的存活率改变 |
| **决策原语 6：早期预警** | 基于状态估计 + 趋势检测，提前 5–15 min 预警隐性失血/失代偿 |
| **解释性输出** | 每一条建议附：(a) 当前关键状态值 (b) 预测轨迹图 (c) 不确定区间 (d) 主导驱动变量 |

## 3. 数据流与接口规范

| 接口 | 方向 | 变量 | 频率 | 说明 |
|---|---|---|---|---|
| L2 → L0 | 下行 | `P_local(t)` | 每个仿真步 (~10 ms) | 损伤位置上游压力，由 L2 解算后从动脉树定位推出 |
| L0 → L2 | 上行 | `Q_bleed(t)` ± σ | 每个仿真步 | 来自降阶代理模型；带不确定区间 |
| L2 → L1 | 下行 | `P_aorta(t)`, 顺应性 | 每个仿真步 | 心室射血波形 + 动脉树参数 |
| L1 → 数据同化 | 上行 | 合成 ABP/PPG/PWV | 每秒 | 用于与实测信号比对 |
| 监护设备 → 数据同化 | 上行 | HR, ABP, SpO₂, EtCO₂, PPG | 1–250 Hz | 实测信号 |
| 数据同化 → L2 | 下行 | 状态/参数修正 | 1–5 s | UKF/EnKF 更新 |
| L2 → 决策层 | 上行 | 状态向量 + 短期预测 | 1 s | 决策原语的输入 |
| 决策层 → L2 | 下行 | 候选 `Q_resus`, `Q_drug`, 干预参数 | 决策周期 (5–60 s) | MPC 在 L2 上做 rollout |
| 医生 → 决策层 | 输入 | 损伤分类、干预记录、覆盖原因 | 事件触发 | 通过移动端/床旁界面 |
| 决策层 → 医生 | 输出 | 建议 + 解释 + 置信度 | 1 s + 事件触发 | 床旁/移动端展示 |

## 4. 分阶段实施路线（建议 3 阶段）

### 阶段 1：可用最小产品 (MVP) — 6 个月

**目标**：在仿真器/历史数据上验证决策回路。

| 任务 | 输出 | 选用工具/文献 |
|---|---|---|
| L2 内核移植 | 可运行的 0D 全身模型，输入出血/复苏率 | Jin & Reifman 2023 / Chalumuri-Bighamian 2024 |
| L0 简化版 | 经验型 `Q_bleed = f(MAP)` 公式（无 3D），按损伤类别查表 | 文献参数 + ATLS 估计表 |
| L1 占位 | 暂不启用，监护数据直接输入 L2 数据同化 | — |
| 数据同化 | UKF 实时更新 V_blood、SVR、心室收缩性 | 标准库 |
| 决策原语 1–3 | 状态评估、短期预测、复苏推荐 | MPC 用 SciPy/CasADi |
| 验证数据集 | PROMMTT 公开数据 + 实验室出血动物数据 | Vodovotz/Cannon 2024 已示范 |
| 形态 | 桌面/服务器端原型 + 模拟病例 | — |

**MVP 验收**：在 50 例回放病例上，模型短期 MAP 预测误差 < 10 mmHg；复苏推荐与专家共识一致率 > 70%。

### 阶段 2：3D-CFD ROM 集成 + 床旁试点 — 12 个月

| 任务 | 输出 |
|---|---|
| L0 真正上线 | 6 类典型损伤的离线 3D-CFD/FSI 数据库 + 神经网络代理模型 |
| L0 干预参数化 | 止血带、敷料、填塞的几何/物理建模 |
| L1 启用 | ADAN 子树 + openBF，与监护波形对齐 |
| 数据同化升级 | EnKF 处理多模态信号 (HR/ABP/PPG/SpO₂) |
| 决策原语 4–6 | 允许性低血压窗口、转运决策、早期预警 |
| 临床界面 | 床旁触摸屏 + 移动端，损伤快速录入 + 建议展示 |
| IRB 试点 | 在 1–2 家创伤中心做观察性研究（不干预临床决策） |

**阶段 2 验收**：观察性研究中决策建议与实际医生决策一致率 > 75%；早期预警提前量平均 ≥ 5 min；FDA 510(k) Pre-Sub 启动。

### 阶段 3：监管与部署 — 18 个月+

| 任务 | 输出 |
|---|---|
| 监管路径 | 按 FDA SaMD / 中国 NMPA 三类医疗器械软件路径申报；Bighamian 2024 已示范 PCLC 设备评估方法学 |
| 临床试验 | 多中心 RCT：CDS 辅助 vs 标准救治，主要终点 24 h 死亡率 / 大量输血量 |
| 院前/战伤拓展 | 移植到救护车/前沿手术队 (FST) 的便携设备；离线模式 |
| 实时性能 | 单步 < 50 ms (CPU)；端到端延迟 < 500 ms |
| 持续学习 | 上线后真实数据回流，定期重训 L0 ROM、再校准 L2 参数先验 |

## 5. 验证策略（CDS 必须严格分级）

| 层级 | 验证内容 | 数据 / 方法 |
|---|---|---|
| **V&V — Level 1** | 各层独立单元验证 | L0：CFD-mesh 收敛性 + ROM 重现误差；L1：与已发表 1D 基准一致；L2：与文献模型逐变量对比 |
| **V&V — Level 2** | 跨层耦合验证 | 端到端在动物出血/复苏数据集 (Bighamian 27 羊 + 12 猪 / Cannon 猪+PROMMTT) 上的预测误差 |
| **V&V — Level 3** | 决策算法验证 | 回放历史病例，决策与实际预后/专家共识比对 |
| **V&V — Level 4** | 可用性 + 安全 | IEC 62366 (可用性) + ISO 14971 (风险管理) + IEC 62304 (软件生命周期) |
| **V&V — Level 5** | 临床效力 | 前瞻性 RCT；死亡率、MTP 用量、ICU 天数为终点 |
| **持续监测** | 上线后性能漂移 | 每月 AUROC 监控、参数分布漂移检测、不良事件回溯 |

## 6. 关键风险与缓解

| 风险 | 缓解 |
|---|---|
| L0 ROM 在训练几何外失效（不常见损伤） | 兜底回到经验公式 + 给出"低置信度"标记，不输出强建议 |
| 监护信号缺失/伪迹 | 数据同化降级运行 (只用可用通道)，明确提示不确定性扩大 |
| L2 参数辨识不稳定 | 多先验 ensemble + 物理先验约束 + 变量饱和 |
| 决策建议被盲从 | 强制可解释性、强制医生确认、关键决策需双重确认 |
| 监管不确定 | 早期 Pre-Sub 沟通；按 FDA SaMD Class IIb-III + AI/ML SPS 路径 |
| 计算资源 (院前/战场) | L0 离线 ROM + L2 编译为 C/Rust + 边缘设备部署；离线模式仅运行 L2+决策层 |

## 8. 一句话总结

> **本路线把"机制研究型数字孪生"裁剪为"决策支持型数字孪生"——保留三大计算层 (L0+L1→L2)，把炎症/凝血/MODS 等不直接影响实时决策的子系统替换为一个明确产出可操作建议的决策层；通过离线 3D-CFD 降阶、在线 0D 实时仿真、模型预测控制 + 数据同化的组合，达成"实时性、可解释、可监管、可临床验证"四个 CDS 必备属性。**
