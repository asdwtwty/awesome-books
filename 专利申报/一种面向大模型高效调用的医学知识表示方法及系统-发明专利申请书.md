## 三、说明书摘要

本发明公开了一种面向大模型高效调用的医学知识表示方法及系统。所述方法包括以下步骤：(1) 将医学知识以 Markdown 加 YAML 源格式与 JSONL 发布格式进行双表示存储，并通过格式转换器实现源格式到发布格式的双向无损转换及质量校验；(2) 按"一卡一任务"原则将医学专业知识拆分为包含结构化字段的最小知识卡片单元；(3) 在卡片之间建立六类轻量关系以支持流程编排和条件触发；(4) 将安全约束从知识正文中分离为独立的规则层，并在调用链的三个阶段进行程序化执行；(5) 构建包含查询解析、混合检索、关系扩展、规则校验和证据溯源的一体化调用链。本发明使医学知识既便于专家维护更新，又适合大语言模型程序化调用，在检索精度、安全保障、流程完整性和证据可追溯性方面显著优于现有的固定长度分块方案，**【修改3】适用于战创伤救护、急诊分诊、临床指南一致性审核及重症监护等高风险临床决策支持场景**。

## 六、说明书

### （一）技术领域

**[0001]** 本发明属于人工智能与医学信息学交叉技术领域，具体涉及一种面向**【修改10】大语言模型**检索增强生成（Retrieval-Augmented Generation, RAG）调用链的医学知识结构化表示、组织、检索与安全校验方法及系统。

### （二）背景技术

**[0002]** 大语言模型（Large Language Model, LLM）在医学问答、临床推理和知识总结等任务上展现出强大能力，但同时存在知识时效性不足、证据不可追溯和幻觉输出等问题，制约了其在高风险医疗场景的应用。检索增强生成（RAG）技术通过引入外部知识库为医疗场景提供了可控性更强的技术路线。

**[0003]** 现有医学 RAG 系统在知识组织层面主要采用固定长度文本分块与基于语义相似度的向量检索方式。该类方法存在以下不足：

(1) 固定长度分块破坏了临床知识的语义完整性，检索到的文本片段往往不能独立表达完整的临床任务信息；

(2) 缺乏对医疗流程时序关系的建模能力，无法表达"步骤 A 必须先于步骤 B"等强时序约束；

(3) 将安全约束（禁忌证、升级条件、优先级规则）嵌入到自然语言正文中，**【修改10】大语言模型**在推理过程中容易遗漏或违反；

(4) 证据追溯仅能到文档级或段落级，无法实现字段级的精确溯源。

**[0004]** 特别地，在高风险临床场景（如战创伤救护、急诊分诊、重症监护等）中，知识调用错误可能直接影响患者生命风险。该类场景具有救治阶段强转移、信息不完整、时间窗短、禁忌条件多、伤员动态变化和证据要求强可追溯等特点。现有通用 RAG 方案无法满足上述应用约束需求。

**[0005]** 因此，需要一种新的医学知识表示方法，将已有的长文档形式知识进行细粒度、结构化组织，使知识既便于医学专家维护，又适合**【修改10】大语言模型**程序化索引、检索、关系扩展、安全校验和证据溯源。

### （三）发明内容 **【修改8】【修改9】**

#### 1. 技术问题

**[0006]** 针对现有技术中医学 RAG 系统存在的语义完整性差、时序关系缺失、安全约束嵌入正文易被违反、证据追溯粒度粗等问题，本发明所要解决的技术问题是：提供一种面向大语言模型高效调用的医学知识表示方法及系统，使医学知识既便于专家以人类可读形式维护，又能被机器以程序化方式精确索引、检索、扩展、校验和溯源，从而在高风险临床决策支持场景中实现可控、可释、可追责的智能问答。

#### 2. 技术方案

**[0007]** 为解决上述技术问题，本发明提供一种面向大模型高效调用的医学知识表示方法，包括如下步骤：

**[0008]** 步骤 S1：双表示知识格式构建。将医学知识对象同时以源格式和发布格式存储；所述源格式采用 Markdown 正文加 YAML 元数据头的形式，用于医学专家的人工维护；所述发布格式采用 JSONL 形式，用于机器索引和程序调用；通过格式转换器实现源格式到发布格式的自动映射，并在转换过程中执行字段完整性校验、术语标准化校验、引用有效性校验和受控词表一致性校验。

**[0009]** 步骤 S2：知识卡片化组织。按"一卡一任务"原则将医学知识拆分为结构感知、语义完整、可独立验证的最小知识单元——知识卡片；每张知识卡片包含唯一标识 ID、类型标签、场景标签、触发条件、操作步骤、禁忌条件、证据来源和版本信息等结构化字段；所述类型标签包括概念卡、评估卡、处置卡、用药卡、装备卡、决策卡和禁忌卡中的一种；所述场景标签包括救治阶段、施救角色、环境类型和严重程度四维标签。

**[0010]** 步骤 S3：轻量关系层建模。在知识卡片之间建立六类轻量关系，包括：严格前置顺序关系 *precedes*、条件触发关系 *triggers*、同类替代关系 *alternative_to*、通用到具体细化关系 *refines_to*、互斥禁忌关系 *contradicts* 和后送转运关系 *escalates_to*；所述关系以 CSV 表结构存储，每条关系记录包含源卡片 ID、关系类型、目标卡片 ID、触发条件、优先级和证据来源；所述触发条件采用受限领域特定语言（DSL）表达，通过抽象语法树解析后静态执行。

**[0011]** 步骤 S4：安全约束外置化规则构建。将禁忌证、优先级和后送条件从知识正文中分离，构建独立的规则层；所述规则层包括三个级别：全局刚性约束、场景条件约束和知识关联约束；每条规则包含规则标识、作用域、严重程度、触发条件表达式、执行动作、警示消息、证据来源和优先级字段；所述严重程度分为绝对阻断 *hard_stop*、强制警示 *warning* 和建议提示 *advisory* 三级。

**[0012]** 步骤 S5：一体化调用链执行。当接收到用户查询时，依次执行如下子步骤：

- S5.1 查询解析与场景识别：从用户输入中提取意图、实体和五维场景标签；
- S5.2 规则层第一阶段执行——检索前预筛选：对高风险查询执行拦截或改写；
- S5.3 混合检索：同时执行基于场景字段的元数据过滤检索、基于 BM25 的稀疏术语检索和基于向量嵌入的稠密语义检索，按预设权重融合得分排序；
- S5.4 关系层扩展：以检索命中卡片为起点，沿 *precedes*、*triggers*、*refines_to* 和 *escalates_to* 四类关系最多扩展预设跳数，补充前置步骤、替代方案和升级路径；
- S5.5 规则层第二阶段执行——检索后过滤：检查候选卡片集合中的互斥冲突，移除违规卡片并注入警示信息；
- S5.6 上下文编排与生成：将通过安全过滤的卡片集合按结构化模板注入大语言模型上下文，要求模型对关键主张标注来源卡片 ID，生成回答；
- S5.7 规则层第三阶段执行——生成后复核：对大语言模型输出进行红旗检测，补充缺失的安全警示，必要时执行最终拦截；
- S5.8 证据溯源与日志存储：为本次调用生成全局追踪标识 *trace_id*，将每个子步骤的输入输出保存。

**[0013]** 进一步地，所述步骤 S2 中，知识卡片 ID 采用层级命名规则"{领域}.{类型}.{主题}.{序号}"，其中领域标识知识来源（如 TCCC 指南、PCC 指南等），类型标识卡片功能（如 *proc*、*assess*、*drug*、*decision*），主题标识具体内容，序号为三位数字编号。

**[0014]** 进一步地，所述步骤 S2 中，知识卡片的粒度判定遵循四条准则：(1) 每张卡片有且仅有一个任务目的；(2) 卡片内所有内容由同一组触发条件触发；(3) 可有多条证据引用但围绕同一主题；(4) 脱离其他卡片，临床专家仍可判断其正确性和完整性。

**[0015]** 进一步地，所述步骤 S3 中，受限领域特定语言的语法规则为：表达式由原子式通过 AND、OR、NOT 逻辑组合构成；原子式格式为"字段名 运算符 值"；运算符包括等于、不等于、大于、小于、大于等于、小于等于、属于和不属于；值类型包括数字、字符串、布尔和列表；解析器须生成抽象语法树后静态执行，禁止使用动态代码执行。

**[0016]** 进一步地，所述步骤 S5.3 中，混合检索的融合公式为：

> *final_score = α × bm25_score + β × dense_score + γ × meta_match_score*

其中 α、β、γ 为可配置权重参数，暴露于配置文件中以支持评测消融。

**[0017]** 进一步地，所述步骤 S5.5 和 S5.7 中，规则层三个阶段使用统一的裁决结构返回，该裁决结构包含裁决结果字段（取值为允许、警告或阻断）、原因列表字段、证据标识列表字段和上下文修改字段。

**[0018]** 进一步地，所述步骤 S1 中，格式转换器在转换过程中执行的校验包括：(a) 卡片 ID 命名符合预定正则表达式；(b) 预设必填字段齐全；(c) 证据来源至少一条且包含来源标识、章节和定位信息；(d) 引用的目标卡片 ID 在知识库中存在；(e) 受控词表字段值在预设枚举值内。校验不通过的卡片在发布阶段被阻断。

**[0019]** 本发明还提供一种面向大模型高效调用的医学知识表示系统，包括：

- 知识仓储模块：存储 Markdown 加 YAML 源格式文件和 JSONL 发布格式文件，维护知识卡片的版本历史；
- 格式转换模块：执行源格式到发布格式的自动转换，并在转换过程中执行上述多项一致性校验；
- 索引与检索模块：对发布格式的知识卡片构建 BM25 稀疏索引、向量稠密索引和元数据索引，支持三路混合检索；
- 关系层模块：维护知识卡片之间的六类轻量关系，执行有限深度的关系扩展和冲突检测；
- 规则层模块：维护三级安全约束规则，在调用链的检索前、检索后和生成后三个阶段独立于大语言模型推理过程执行安全校验；
- 调用编排模块：接收用户查询后，依次调度场景识别、规则预筛、混合检索、关系扩展、规则过滤、大语言模型生成和生成后复核各子模块，并为全过程生成追踪标识和日志存储；
- 可观测层模块：为每次调用保留来源卡片标识、关系扩展路径、触发规则清单和各步骤执行日志，支持任意调用的完整重放和证据追溯。

#### 3. 有益效果 **【修改9】**

**[0020]** 与现有技术相比，本发明可应用于战创伤救护问答与辅助决策、急诊分诊辅助决策、临床指南一致性审核、医学培训考核及重症监护决策支持等场景，并具有以下显著优点：

**[0021]** (1) **知识维护与调用解耦**：源格式采用 Markdown，医学专家无需编程知识即可维护知识卡片内容；发布格式采用 JSONL，机器系统可直接索引和调用。双格式双向无损转换，使知识维护和系统开发可独立并行进行。

**[0022]** (2) **检索精度显著提升**：知识卡片以临床任务为边界，比固定长度分块具有更高的语义完整性；三路混合检索兼顾专业术语缩写、自然语言描述和场景条件过滤。

**[0023]** (3) **临床流程完整性保障**：轻量关系层通过六类关系建模流程时序、条件触发和升级路径，使大语言模型生成的流程回答保留正确顺序。

**[0024]** (4) **高风险场景安全性显著增强**：安全约束以程序化规则独立于模型推理过程执行，三阶段多道防线确保禁忌条件不被遗漏。

**[0025]** (5) **工程可落地性强**：统一的字段模式、明确的命名规则、CI 强制校验流水线和标准化评测框架，使该方法从论文概念变为可调用知识的工程实现。

### （五）具体实施方式 **【修改13】**

**[0031]** 下面以战创伤早期救治为例，结合附图对本发明作进一步详细说明。所述实施例仅用于解释本发明，不用于限定本发明的保护范围。

#### 实施例 1：知识卡片双表示格式

**[0032]** 如图 2 所示，知识卡片存在两种形态。**【修改11】**

源格式（用于人工审核维护）示例如下：

```yaml
card_id: tccc.proc.tourniquet.001
type: procedure
title: "战术现场护理阶段肢体止血带应用"
scenario:
  phase: TFC
  role: [combat_medic, cls]
severity: life_threatening
triggers:
  - "可见的危及生命的肢体外出血"
evidence:
  - source: "TCCC_2026"
    section: "Tactical Field Care §3.a"
    page: 2
version: "2026.05"
status: published
---
## 处置步骤
1. 选用 CoTCCC 推荐型止血带
2. 直接置于皮肤上，距出血点近端 5–7 cm
3. 收紧至出血停止且远端搏动消失
```

发布格式（用于机器调用）示例如下：

```json
{
  "card_id": "tccc.proc.tourniquet.001",
  "type": "procedure",
  "actions": [
    {"order": 1, "text": "选用 CoTCCC 推荐型止血带"},
    {"order": 2, "text": "直接置于皮肤上，距出血点近端 5–7 cm"},
    {"order": 3, "text": "收紧至出血停止且远端搏动消失"}
  ],
  "contraindications": ["关节上方", "眼眶 1 cm 范围内"],
  "evidence": [{"source": "TCCC_2026", "section": "§TFC.3.a", "page": 2}],
  "embedding_ref": "emb/v1/tccc.proc.tourniquet.001.bin"
}
```

**[0033]** 格式转换器在转换过程中执行五项校验：ID 命名合规、必填字段完整、证据来源有效、引用目标存在和词表一致性。任一项不通过则阻断发布。本实施例表明，源格式与发布格式之间可实现双向无损映射，并显著降低知识维护门槛。

#### 实施例 2：轻量关系层

**[0034]** 关系层以 CSV 表结构存储，示例如下：

```csv
source_id,relation_type,target_id,condition,priority,evidence_id
tccc.assess.airway.001,precedes,tccc.proc.cricothyroidotomy.001,,1,TCCC_2026_§4
tccc.proc.tourniquet.001,escalates_to,tccc.decision.tq_conversion.001,"duration_min > 120 AND has_monitoring == false",2,TCCC_2026_§Circ.a.3
tccc.assess.tension_pneumo.001,triggers,tccc.proc.needle_decompression.001,"resp_distress == true OR sbp < 90",1,TCCC_2026_§Resp.a.2
```

**[0035]** 关系扩展器以检索命中卡片为种子，沿四类正向关系（*precedes*、*triggers*、*refines_to*、*escalates_to*）向前后各最多扩展 2 跳。扩展结果与原始检索结果合并后，再经规则层过滤。条件表达式采用受限 DSL，其语法规则如下：

```ebnf
expr  := atom | atom AND expr | atom OR expr | NOT atom
atom  := field op value
field := 下划线风格标识符
op    := == | != | > | < | >= | <= | in | not_in
value := 数字 | 字符串 | 布尔 | 列表
```

解析器先将表达式解析为抽象语法树，然后静态执行，禁止 `eval` 等动态代码执行。

#### 实施例 3：安全约束规则层三阶段执行

**[0036]** 如图 4 所示，规则层在调用链的三个阶段独立执行：

- **第一阶段（检索前预筛）**：输入为查询上下文。识别高风险查询（如寻求违反禁忌的操作指导），在检索前即执行拦截或查询改写。
- **第二阶段（检索后过滤）**：输入为查询上下文和候选卡片集合。利用 *contradicts* 关系和规则条件检测候选集中的互斥冲突，移除违规卡片，注入 *warning* 信息到上下文。
- **第三阶段（生成后复核）**：输入为查询上下文和大语言模型输出文本。对回答内容进行红旗检测，补充缺失的安全警示，对违规内容执行最终拦截。

**[0037]** 三个阶段使用统一的裁决结构（Verdict）返回：

```json
{
  "verdict": "allow | warn | block",
  "reasons": [{"rule_id": "...", "severity": "...", "message": "..."}],
  "evidence_ids": ["..."],
  "mutated_context": {"add": [...], "drop": [...]}
}
```

**[0038]** 优先级仲裁规则：并发触发时按 *priority* 数值升序裁决；*hard_stop* 永远优先于 *warning*，*warning* 优先于 *advisory*。

#### 实施例 4：混合检索三路融合

**[0039]** 如图 5 所示，混合检索由三部分组成：

(1) **元数据过滤检索**：基于知识卡片的 *scenario.phase*、*type*、*severity* 等结构化字段进行精确匹配过滤，缩小候选范围。

(2) **BM25 稀疏检索**：对卡片正文建立倒排索引，捕获医学领域中大量缩写（TQ、NPA、NDC、IO、EtCO₂ 等）和标准术语的精确匹配。

(3) **向量稠密检索**：使用多语言句子编码器（如 *multilingual-e5-base*）对卡片正文编码为稠密向量，覆盖自然语言症状描述和语义近似查询。

**[0040]** 三路得分按融合公式加权合并：

> *final_score = α × bm25_score + β × dense_score + γ × meta_match_score*

默认参数 α=0.5、β=0.4、γ=0.1，所有权重在配置文件中暴露，支持消融评测。本实施例表明，三路融合策略相对于纯稠密检索可在医学缩写匹配与场景过滤上获得显著提升。
