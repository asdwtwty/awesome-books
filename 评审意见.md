# 青年科学基金项目申报材料评审意见

**项目名称：** 机理－数据双驱动的创伤失血性休克伤员数字孪生关键技术研究
**评审定位：** 相关领域（生物医学工程 / 医学人工智能 / 生理系统建模）专家函评
**评审目的：** 指出材料中影响立项的问题，给出可操作的修改建议，提高申报命中率

---

## 一、总体评价

选题面向"创伤失血性休克（THS）院前早期救治"这一国家急需且具显著社会效益的方向，结合数字孪生、机理建模与人工智能等前沿手段，**立意新、应用前景明确、与国家院前急救体系建设和"虚拟生理人体"前沿契合度高**，整体具备良好的资助潜力。

但作为青年科学基金（强调"一个明确、可深入的核心科学问题 + 申请人独立完成能力 + 三年可交付"）的申报材料，当前版本存在以下**三类结构性问题**，是影响命中率的主要风险点，需重点修改：

1. **贪大求全、重点不突出（最关键）**：同时铺开"0D–1D–3D 多分辨率切换 + 循环-呼吸-神经-内分泌四系统耦合 + 机理-数据混合建模 + 在线参数辨识 + 边缘端原型平台 + 多源临床验证"，体量接近面上/重点项目，青年基金体量（一般 1 名负责人、3 年、经费有限）难以全部完成，评审极易判定"可行性不足、科学问题分散"。
2. **科学问题与工程目标混淆**：三个"关键科学问题"中混入了"计算实时性""可落地"等工程指标，未提炼为可证伪的科学命题，削弱了"基础研究"属性。
3. **可行性论证存在硬伤**：混合建模的**训练标签来源**、稀疏观测与多参数辨识的**内在矛盾**、3D 个性化所需**影像来源**、隐状态（失血量/外周阻力）的**验证金标准**等关键环节论证不足；且材料缺少"研究基础与申请人简历"，而青年基金高度依赖该部分。

下文按模块给出具体问题（标【问题】）与修改建议（标【建议】），并在第四节给出按优先级排序的 Top 修改清单。修改后的全文见同目录 `申报书_修改版.md`（改动以【增】【删】【改】标注）。

## 二、主要优点（建议保持）

- 临床痛点（黄金 1 小时、代偿期隐匿性出血、个体差异、院前监测受限）凝练准确，与政策背景（突发事件创伤救治规范、院前院内信息链接标准、十五五健康数智化）结合紧密。
- 国内外现状覆盖面广、文献较新（2024–2026 多篇），并已注意到主流仿真引擎（HumMod/BioGears/Pulse）在大失血场景的偏差，论证了"专用建模"的必要性。
- "机理 + 数据 + 可穿戴"三位一体技术路线方向正确；研究目标区分科学/技术/应用三层，结构清晰。
- 已有前期积累（0D 闭环建模 Python 实现、可穿戴多参数采集、专利），具备开展条件。

## 三、分模块问题与修改建议

### （一）摘要

- 【问题1】**字数与一致性**：摘要接近/可能超过 400 字上限，需核减；且摘要中"多生理系统协同优化耦合"与正文研究内容(2)"融合机理约束的深度学习建模"、关键科学问题(2)"机理—数据可解释融合"三处措辞不一致，读者难以建立"科学问题—研究内容—目标"的对应。
- 【问题2】**创新点不显**："预期形成自主知识产权的原型系统"偏工程，未点明方法学层面的"新"在哪里。
- 【建议】统一三处表述为同一套术语（建议：①多分辨率血流动力学统一表征；②机理-数据双驱动多系统耦合建模；③稀疏观测个性化参数辨识与不确定性量化）；删减背景性文字，把节省的篇幅用于一句话点明方法创新（如"提出守恒约束下的多分辨率切换接口"与"可微端到端的机理-残差混合建模"）。压缩至 380–400 字以内。

### （二）立项依据（研究意义、现状）

- 【问题3】**批判性与"靶点"不够聚焦**：现状综述信息量大，但"每类方法的局限 → 本项目如何针对性解决"的逻辑链偏弱，三个"瓶颈"与后文三个科学问题、三项研究内容未显式对应。
- 【问题4】**文字硬伤**：
  - "心血管系统涉及动脉、静脉、毛细血管等**个多尺度**" → 应为"等**多个尺度**"。
  - "用于**描述描述**心脏、血管和肺……"（3.2 节）重复"描述"。
  - "为本项目开展……**的提供了**方法学储备" → 多一个"的"。
  - "美国国防部**今年启动**了……VITAL 项目" → 应写明确切年份（如 2025 年/2026 年），避免"今年"这类随时间失效的表述。
- 【问题5】**政策引用核实**："国家十五五规划纲要"建议核实正式名称与发布时间（规划《纲要》通常于 2026 年全国两会审议通过），避免名称不规范被质疑。
- 【建议】在 1.2 节末尾保留三点瓶颈，但改写为与科学问题严格对应的形式，并在每条后用一句指明"本项目对策"。补一句"研究基础"的指引（见可行性部分意见）。

### （三）研究目标

- 【问题6】**科学目标偏窄**：科学目标只覆盖"参数可辨识性边界"（对应科学问题3），未覆盖科学问题1（多尺度统一表征）和2（机理-数据融合），目标与关键科学问题不对齐。
- 【建议】将科学目标改写为覆盖核心科学问题的"主-辅"结构，或明确"本项目以科学问题3为核心、问题1/2为支撑"，使目标—问题—内容三者一致。

### （四）拟解决的关键科学问题（核心修改区）

- 【问题7】**混入工程指标**：科学问题(1)"……兼顾高保真与**计算实时性**"，"计算实时性"属工程性能而非科学命题；科学问题(3)"建立快速、稳定的个性化模型参数**估计方法**"亦偏方法/技术。
- 【问题8】**问题偏多、彼此重叠**：青年基金通常建议聚焦 1（至多 2）个核心科学问题；当前(1)与(2)均落在"统一框架/统一损失"，存在重叠。
- 【建议（重要）】**收敛为一个核心科学问题 + 至多一个支撑性问题**。推荐以下提炼：
  - **核心科学问题**："在院前稀疏、含噪生命体征观测条件下，创伤失血性休克多系统耦合机理模型关键参数的**结构/数值可辨识性边界**是什么，以及如何在该边界内实现可量化不确定性的个性化反演与失代偿前预警。"（可证伪、可深入、契合青年基金）
  - **支撑性科学问题**："多分辨率血流动力学模型在分辨率切换界面上，满足质量/动量守恒的**误差可控传递条件**。"
  - 将"机理-数据如何融合""实时性"等下沉到研究方案/技术目标中表述，不再列为科学问题。

### （五）研究内容

- 【问题9】**体量过大、主次不分**：三项内容平行铺开，缺"核心—扩展"层次；与青年基金体量不匹配。
- 【问题10】**内容与科学问题不一一对应**。
- 【建议】重组为"一条主线 + 有限扩展"：
  - 主线（核心，占 ~60% 工作量）：内容(3) 机理-数据双驱动的稀疏观测个性化参数辨识与不确定性量化（对应核心科学问题）。
  - 支撑1：内容(1) 以 0D 全身闭环为核心、1D 为有限增强的多分辨率建模；**3D 降为"离线模板库 + 降阶代理"的可选支撑，明确不作为青年基金主交付**。
  - 支撑2：内容(2) 机理-残差混合建模，**建议将"四系统全耦合"收敛为"循环-呼吸为主、神经反射与内分泌以低阶反馈项纳入"**，降低标注与辨识负担。

### （六）研究方案与技术路线（可行性硬伤集中区）

- 【问题11（重大）】**混合模型训练标签来源不明**：用 Neural ODE + 残差网络拟合"毛细血管渗漏系数、激素释放率、神经反射增益"等残差动力学，但这些量**临床无法直接测量、缺乏监督标签**，材料未说明监督信号从何而来，方法存在不可训练风险。
  - 【建议】明确"**机理模型/虚拟人群生成合成轨迹做预训练 → 少量脱敏真实数据微调 → 物理/生理约束做自监督/半监督正则**"的三段式训练范式；并说明合成数据的覆盖性与域偏移（sim-to-real）应对。
- 【问题12（重大）】**稀疏观测 vs 多参数辨识的内在矛盾**：院前仅 HR/BP/SpO₂/RR/EtCO₂ 等少量观测，却要辨识多系统数十参数，UKF 极易发散或陷入不可辨识。
  - 【建议】明确"**可辨识参数子集规模上限与观测维度的定量关系**"，给出结构可辨识性分析的具体工具（如基于灵敏度矩阵秩、Profile Likelihood、微分代数方法），并设定 Top-K 的 K 值范围与冻结策略；这恰是核心科学问题，应重点论证而非一笔带过。
- 【问题13】**3D 个性化的影像来源逻辑漏洞**：院前无 CT，3D 靠"离线影像重建 + 降阶代理"。但"个性化"3D 需患者本人影像，离线用的是谁的影像？
  - 【建议】明确使用"**代表性损伤部位模板库 / 统计形状模型 (SSM)**"而非个体即时 CT，定位为"群体级先验 + 在线 0D 修正"，避免逻辑矛盾。
- 【问题14】**多分辨率"切换准则"不具体**：何种阈值/事件触发 1D/3D，切换时状态如何初始化/插值，误差传递如何度量，均缺定量描述。
  - 【建议】给出触发条件（如"压迫止血/复苏事件 + 波形需求"）、界面守恒约束的离散化方式、切换瞬间状态映射与误差校验指标。
- 【问题15】**验证数据集匹配度存疑**：MIMIC-IV（ICU、非院前、创伤占比低）、PROMMTT（创伤大出血输血，院前连续变量有限）、PhysioNet（泛指）。这些数据能否提供模型所需的连续多系统变量及隐状态"真值"（失血量、外周阻力）存疑。
  - 【建议】区分"**可观测量预测精度**（HR/BP/SpO₂ 等，有真值）"与"**隐状态估计**（无金标准，用机理一致性 + 受控动物/文献数据 + 趋势/事件预测间接验证）"两条验证路径，并坦承隐状态验证的局限；明确各数据集对应验证哪一类指标。
- 【问题16】**量化指标缺依据**："误差率≤5%""一致性≥70%""RMSE"阈值未给出处。
  - 【建议】给出阈值来源（文献/临床可接受范围）或改为"相对基线方法的改进幅度"。
- 【问题17】**伦理与合规**：100 例脱敏临床数据未写明伦理审批进度、数据来源单位、知情同意/豁免、数据安全与去标识方案。
  - 【建议】补充伦理审批与合作医院数据可获得性证明（如有意向协议更佳）。
- 【问题18】**UKF 之外缺备选与风险对策**：强非线性高维下 UKF 可能不足。
  - 【建议】补充备选（集合卡尔曼 EnKF、粒子滤波、变分推断/集体变分推断[30]）与切换判据，形成风险对策。

### （七）可行性分析

- 【问题19（重要）】**研究基础与申请人简历缺失**：正文多处"详见研究基础部分"，但材料未包含研究基础、申请人代表作与简历；青年基金对申请人独立研究能力与前期积累极为看重。
  - 【建议】补充：申请人近年代表性论文/专利（点明专利[17]系申请人成果）、与本项目直接相关的预研结果（建议附 0D 模型已实现的仿真曲线图、与文献对照的初步验证），以体现"已经走通了关键一小步"。
- 【问题20】**"团队"表述与青年基金定位**：青年基金以申请人个人为主，"团队涵盖临床/工程/AI/嵌入式"应改写为"申请人具备 X 能力 + 依托单位/合作单位提供 Y 支撑"。

### （八）特色与创新

- 【问题21】**措辞与可辩护性**：创新点表述总体可，但应避免"首次/首个"等绝对化用语（易被同行举反例）。建议用"提出/构建/建立"。
- 【建议】将创新点与核心科学问题强绑定，突出"稀疏观测下可辨识性边界 + 可量化不确定性的个性化反演"这一最具基础研究价值的点。

### （九）年度计划与预期成果

- 【问题22】**年度计划过载、缺里程碑与风险节点**：第二年同时完成"1D + 3D + 多分辨率接口 + 参数估计"，过密，且无阶段可交付物与 Go/No-go 判据。
  - 【建议】重排为"第1年：0D 核心 + 出血/复苏 + 个性化辨识雏形并跑通真实数据；第2年：1D 增强 + 机理-残差混合 + 可辨识性边界理论；第3年：原型集成 + 验证 + 不确定性量化 + 结题"，3D 模板库作为弹性任务；每年给出明确交付物与风险对策表。
- 【问题23】**成果指标偏泛**："3–5 篇 SCI/EI"建议明确 SCI 篇数与方向（如至少 2 篇 SCI，覆盖建模方法与辨识方法各一）。

### （十）文字、引用与图表规范

- 【问题24】**图缺失**：图1–图4 仅有标题无图，正式提交版必须补全示意图（系统架构、多分辨率架构、混合建模、参数估计流程）。
- 【问题25】**编号符号统一**："⑴⑵⑶""(1)(2)(3)"混用，建议全文统一。
- 【问题26】**休克定义细节**：可补充 SI 正常参考范围（约 0.5–0.7）及其分级阈值，使"粗略评估"更严谨。
- 【问题27】**参考文献**：总体规范、较新；建议核对中英文作者缩写一致性与个别条目（如[13]卷期信息完整性、[10] DARPA 文件编号与年份）。

## 五、结论

本项目方向好、潜力大，**主要短板在"聚焦"与"可行性论证"，而非选题本身**。建议按上表 Top 1–4 做"伤筋动骨"的聚焦式修改（收敛科学问题、重组内容、补研究基础、说清训练范式），再完成 5–10 的完善，命中率将明显提升。修改后全文见 `申报书_修改版.md`。
