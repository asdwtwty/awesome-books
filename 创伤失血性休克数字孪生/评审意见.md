# 青年科学基金申报材料评审意见

**项目名称**：机理－数据双驱动的创伤失血性休克伤员数字孪生关键技术研究
**评审日期**：2026 年 5 月
**评审定位**：青年科学基金项目（强调创新性、可行性、申请人独立承担研究的能力）

---

## 一、总体评价

本项目选题契合"健康中国 2030"和国家"十五五"规划中关于院前急救与健康数智化建设的方向，瞄准创伤失血性休克这一"可预防性死亡"首要因素，将数字孪生与机理－数据双驱动建模相结合，**选题具有明确的临床意义与前沿性**。整体框架清晰、逻辑较完整，研究内容涵盖多分辨率血流动力学建模、机理－数据混合建模、个性化参数辨识三个层次，与拟解决的三个关键科学问题对应良好。

但存在以下需要重点改进之处，按重要程度排序：

| 序号 | 问题类别 | 严重程度 | 影响 |
| ---- | -------- | -------- | ---- |
| 1 | 错别字与术语不规范（"失血情休克"/"数据孪生"/"中职技术人员"等） | ★★★ | 直接影响评审第一印象 |
| 2 | 文献时效性与引用规范（引用日期出现 2026 年文献，需核对） | ★★★ | 评审会质疑学术严谨性 |
| 3 | 关键科学问题与研究内容创新性凝练不够聚焦 | ★★★ | 影响立项必要性判断 |
| 4 | 可行性分析不充分，研究基础描述薄弱 | ★★★ | 青年基金尤其看重 |
| 5 | 考核指标偏高，与青年基金体量不完全匹配 | ★★ | 易被认为预期过满 |
| 6 | 数据来源、伦理与数据安全声明缺失 | ★★ | 涉及临床数据必须说明 |
| 7 | 年度计划过于简略，缺少里程碑节点 | ★★ | 影响过程可控性评估 |
| 8 | 国际合作与学术交流计划未单独说明 | ★ | 标题要求需对应作答 |
| 9 | 部分图表标题、表格内容可读性不足 | ★ | 影响评审阅读体验 |

## 二、分项问题与修改意见

### 2.1 摘要（限 400 字）

**存在问题**：

1. 现版字数约 410 字（含标点），略超 400 字限制。
2. 数字孪生作为核心概念缺少一句话定义，非建模背景的评审专家可能不易把握。
3. "三大科学问题"表述与正文 2.3 节略有不一致（正文使用"统一数学框架"等）。
4. 摘要末句"提供技术支撑"较空泛，建议落到具体能力（"实时监测—预测—模拟"三位一体）。
5. 关键词"个性化参数辨识"建议改为更标准的"数据同化"或"在线参数辨识"。

**修改建议**：见 `申报书_修改版.md` 摘要部分。

### 2.2 关键词

- 建议改为：`创伤失血性休克；数字孪生；多分辨率血流动力学；机理-数据混合建模；在线参数辨识`
- 删除并列重复，使关键词覆盖三个研究内容。

### 2.3 考核指标

**存在问题**：

1. **错别字**："培养硕士研究生、**中职技术人员** 2–3 名" —— 应为 **工程技术人员** 或 **博士研究生**。
2. 指标量级对青年基金略偏高（青年基金通常论文 2–4 篇、专利 1–2 项即可），建议适度收敛，给评审"留白"。
3. "原型系统 1 套"过于笼统，建议补充功能描述（如：覆盖在线建模、伤情预测、干预模拟三项核心功能）。
4. 学术会议次数与培养人才数量可合并表述。

**修改建议**：

```
- 发表 SCI 检索论文 2–4 篇（其中 1 区或 Top 期刊不少于 1 篇）；
- 申请国家发明专利 1–2 项；
- 登记软件著作权 1 项；
- 研发数字孪生伤员原型系统 1 套（具备在线建模、生理演化预测、复苏方案模拟三项核心功能）；
- 参加国内/国际学术会议交流不少于 2 次；
- 培养硕士研究生 2 名、协助培养博士研究生 1 名。
```

### 2.4 立项依据（1.1 / 1.2）

**存在问题**：

1. **文献时效性**：参考文献中 [6] [9] [10] [26] 标注 2026 年访问/发表，需核对发表时间是否真实，避免使用尚未公开的文献。
2. **学位论文比例偏高**：[14][15][16] 均为硕士学位论文，建议替换为同主题的期刊文献（如 IEEE TBME、Journal of Trauma 等）。
3. **国内外研究现状结论性段落** 1.2.3 末尾"主要存在以下三方面瓶颈"与拟解决科学问题对应不够紧密，建议明确"现有瓶颈—本项目突破点"的对照。
4. **错别字**：
   - 1.2 节末尾"失血**情**休克"多处出现 → "失血**性**休克"；
   - 3.(3) 节"**数据孪生**本质是基于真实测量值实时调整对应的模型参数" → "**数字孪生**"。
5. 1.1 节"申请人团队前期已在……三方面进行了探索（详见研究基础部分）" —— 但全文未见独立的研究基础章节，需要补写或与立项依据合并说明。
6. 1.2.2 节关于 0D/1D/3D 模型的论述较好，但缺少"为何要切换"的临床动机阐述，应补充：救治阶段不同 → 时空尺度需求不同 → 切换是必然选择。

**修改建议**：

- 替换或删除 [14][15][16] 三篇学位论文，改用以下方向的期刊文献：
   - 院前创伤大数据预测预警（如 J Trauma Acute Care Surg、Crit Care）；
   - 可穿戴/无创血流动力学监测（如 IEEE J Biomed Health Inform）。
- 在 1.2 节末尾增加一段"本项目对应的突破点"，与 2.3 节关键科学问题一一对应。
- 增补一段"申请人前期工作基础"（即使内容不长），说明在心血管 0D 闭环建模、可穿戴信号采集、急救信息系统三方面的成果与积累，可包含：相关论文/专利/软件/数据集，以增强青年基金评审的可行性判断。

### 2.5 研究内容、研究目标与关键科学问题（2.1–2.3）

**存在问题**：

1. 科学目标与技术目标存在交叉，建议拉开层次：
   - 科学目标：揭示规律、建立理论；
   - 技术目标：突破关键技术；
   - 应用目标：形成原型系统。
2. 关键科学问题表述过长，每条应控制在一句话内，并与研究内容严格对应。
3. 研究内容(2)"融合机理约束的深度学习失血性休克建模"的"机理-AI 端到端联合训练"描述不够具体，需要明确：是否使用神经 ODE？PINN？还是机理引导的残差网络？
4. 缺少系统架构图（图 1）的简要说明文字。

**修改建议**：见 `申报书_修改版.md` 第二部分。

### 2.6 研究方案与技术路线（第 3 节）

**存在问题**：

1. **可行性分析缺失**：标题写明"包括……可行性分析"，但正文几乎只有研究方法，无独立可行性分析段落。建议在第 3 节末尾增加"3.5 可行性分析"，从理论可行性、技术可行性、数据可行性、团队可行性四个维度论证。
2. **表 1 内容**：
   - 表头"急救适用场景"描述过于简单，建议补充各维度模型的"使用触发条件"；
   - "1D 简单 Navier-Stokes 方程"应为"一维 Navier-Stokes 方程（含血管壁弹性约束）"；
   - "3D 计算流体动力学、流固耦合"中的"血管壁应用"应为"血管壁应力"（错别字）。
3. **测试与验证**：
   - "项目申请单位及合作医院的脱敏临床数据，创伤失血性休克不少于 1000 例" —— 1000 例对青年基金而言体量较大且需要明确数据来源、伦理审批、数据共享协议；
   - 公开数据集中 **MIMIC-IV** 主要为 ICU 数据、**PhysioNet** 没有专门的"院前急救"数据集（主要是 PhysioNet Challenge 中的部分数据），需要核对数据来源真实性；
   - PROMMTT 队列是公开的（PRospective Observational Multicenter Major Trauma Transfusion），可以保留；建议补充 NTDB（National Trauma Data Bank）或 TARN（英国创伤审计研究网络）。
4. **个性化参数估计**：
   - UKF 算法虽适合非线性，但对于循环-呼吸-神经-内分泌耦合系统的高维状态变量，单纯 UKF 可能存在维度灾难问题；建议提及备选方案（集合卡尔曼滤波 EnKF、变分贝叶斯）。
5. **图 2/3/4 仅有图题缺少图本体**，且图与正文叙述顺序不一致。在 Markdown 文档中无法绘图，但应在正文中以文字补充每张图的关键信息要点。

**修改建议**：

- 增加 3.5 可行性分析；
- 修订表 1；
- 缩减并明确数据来源（如：自建队列 100–200 例 + 公开队列），并声明伦理审批；
- 在参数辨识章节增加"算法降维与备选方案"的论述。

### 2.7 特色与创新（第 4 节）

**存在问题**：

1. 三条创新点中"理论创新"与"方法创新"边界不够清晰，"方法创新"实际上同时包含了 2 条研究内容（混合建模 + 多分辨率切换），可拆分。
2. 未直接对比国内外类似项目（如 DARPA VITAL、欧盟 EDITH），无法体现差异化优势。

**修改建议**：

- 创新点改为"科学问题创新 / 关键技术创新 / 临床应用创新"三层结构；
- 每条创新明确"现有方法的局限—本项目的突破—预期带来的差异"。

### 2.8 年度研究计划与预期结果（第 5 节）

**存在问题**：

1. 时间起点应为获批后开始（一般为 2027.01 而非 2026.12），与近期国家自然科学基金时间表保持一致。
2. 三年计划过于宏观，缺少季度/半年里程碑、阶段性成果（论文、专利、软件登记）安排。
3. 未单独列出"国际合作与交流计划"。
4. 未列出"预期研究结果"清单与考核指标的对应关系。

**修改建议**：见 `申报书_修改版.md` 第五部分（已重新撰写）。

### 2.9 其他规范性问题

1. **错别字与术语不统一**（请全文修订）：
   - "失血**情**休克" → "失血**性**休克"（多处）；
   - "**数据孪生**" → "**数字孪生**"（3.(3) 节）；
   - "**血管壁应用**" → "**血管壁应力**"（表 1）；
   - "**中职技术人员**" → "**工程技术人员/博士研究生**"（考核指标）；
   - "描述描述" 重复（3.(2) 节正文）；
   - 半角连字符 `-` 与全角破折号 `—`、en dash `–` 混用，全文应统一为 `—`（破折号）或 `–`（en dash）；
   - 中英文混排时数字、单位前后空格不一致。
2. **格式问题**：
   - 部分公式、术语首次出现未给出英文全称；
   - 参考文献格式未统一（中文文献、英文文献、专利、网页四类，建议按 GB/T 7714 排版）。
3. **图表问题**：
   - 图 1 总体架构图、图 2 多分辨率心血管建模架构、图 3 机理-数据融合多系统建模、图 4 个性化参数估计流程，均仅有图题；建议在最终申报版本中以矢量图（PDF/EPS）形式提交，并在每张图下加一句注释。
4. **伦理与数据安全**：涉及临床数据必须声明：① 已通过/将通过医学伦理审批；② 数据脱敏方案；③ 敏感数据本地化存储与访问控制策略。

## 三、提升立项成功率的关键建议

1. **聚焦"机理-数据双驱动"这一核心创新点**，全文反复回到这一主线，避免被"数字孪生"这一上位概念稀释。
2. **强化前期工作基础**，青年基金评审尤其看重"申请人能不能独立完成"，建议显著增加前期成果展示（论文、专利、软件、合作医院、数据基础）。
3. **凝练 3 条关键科学问题**，每条用一句话表述，并直接映射到研究内容；避免"既要……又要……"的并列长句。
4. **可行性分析独立成节**，从理论、方法、数据、团队、设备五方面论证。
5. **考核指标适度收敛**，留出 20% 上浮空间，避免被认为"画饼"。
6. **统一术语与文献规范**，全文做一次错别字与格式校对。
7. **数据来源与伦理声明**显式给出，体现规范性。
8. **国际合作与学术交流单独成段**，与年度计划呼应。
